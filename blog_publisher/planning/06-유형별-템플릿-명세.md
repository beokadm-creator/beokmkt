# [06] 유형별 콘텐츠 템플릿 명세

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `llm/prompts.py`, `pipeline/generate.py`, `pipeline/factcheck.py`
- 선행 문서: [05](./05-근거기반-콘텐츠엔진-재설계.md)

## 1. 배경

"제대로 된 글"의 필수 구성요소는 유형마다 다르다. 발행 주력은 **리뷰 / 정보성(하우투) / 니치 전문** 세 가지. 각 유형의 필수 섹션·근거 요건·검수 추가 기준을 확정해, 개요 단계가 유형 템플릿을 따르게 한다.

## 2. 공통 규칙 (모든 유형)

- 개요는 05 §4.2의 coverage_targets를 반드시 포함한다(검색 의도 충족).
- 모든 구체 주장에 근거팩 출처(05 §6). 미근거 구체 주장 0건.
- 도입부: 누구의 어떤 문제를 푸는 글인지 2~3문장.
- 마무리: 핵심 요약 + 다음 행동(과장 금지).
- 메타: `meta_description`(80~120자), 추천 내부링크 키워드 2~3개를 산출물에 포함.

## 3. 유형별 명세

### 3.1 리뷰 (review)

검색 의도: "이거 살까? 어떤 게 나은가?" — 구매 의사결정.

필수 섹션:
1. 한줄 평가 + 추천 대상(누구에게)
2. 핵심 스펙/사양 표 (근거팩 facts 기반, 출처 필수)
3. 실사용/평가 포인트 (장점)
4. 단점·주의점 (균형 — 단점 없는 리뷰는 신뢰 하락)
5. 대안 비교 (경쟁 제품 1~2개와 차이)
6. 최종 결론 (어떤 상황이면 추천/비추천)

추가 검수 기준: 장점만 나열 금지(단점 1개 이상), 스펙 수치 출처 매칭, 가격/모델명 등 변동 정보는 confidence·출처 표기.

### 3.2 정보성/하우투 (howto)

검색 의도: "어떻게 하지? 방법이 뭐지?" — 문제 해결.

필수 섹션:
1. 한 줄 요약(결론부터) + 누구를 위한 글
2. 사전 준비물/전제조건
3. 단계별 절차 (순서·번호, 각 단계 구체 행동)
4. 흔한 실수/주의사항
5. (해당 시) FAQ — PAA/연관질문 기반
6. 요약 체크리스트

추가 검수 기준: 단계가 실제로 실행 가능(모호한 일반론 금지), 수치·기준은 출처, 단계 누락/순서 오류 점검.

### 3.3 특정 니치 전문 (niche)

검색 의도: 특정 분야 심화 정보 — 전문성/신뢰(E-E-A-T) 중시.

필수 섹션:
1. 핵심 정의/맥락 (왜 중요한가)
2. 심화 내용 (분야 고유 디테일, 표면적 설명 금지)
3. 근거·데이터 (통계·연구·1차 출처 비중 높게)
4. 실무 적용/시사점
5. 전문가 관점/흔한 오해 정정
6. 출처/참고 (근거 투명성)

추가 검수 기준: 표면적 일반론 비율 낮을 것(니치 깊이), 1차 출처 비중, 전문 용어 정확성.

## 4. 골격 반영

- `prompts.py`: `OUTLINE_TEMPLATES = {"review":..., "howto":..., "niche":...}` 로 유형별 개요 지시 + 필수 섹션 강제.
- `generate.py`: `post.content_type`에 따라 템플릿 선택.
- `factcheck.py`/`review.py`: 유형별 추가 검수 기준 훅(초기엔 공통 + 유형 메모, 점진 강화).

## 5. 미해결 / 다음 단계

- 리뷰의 가격·재고 등 휘발성 정보 갱신 주기 정책.
- 니치 분야 화이트리스트 출처 구성(05 §8과 연계).
- 유형 자동 판별(주제만으로 유형 추론) vs 시드에 유형 명시 — 현재는 시드에 명시 가정.
