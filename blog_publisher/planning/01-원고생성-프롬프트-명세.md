# [01] 원고 생성 프롬프트 명세

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `llm/prompts.py`, `pipeline/generate.py`, `config.py`
- 선행 문서: [DESIGN.md](../DESIGN.md) §2

## 1. 배경 / 문제

기존 방식은 thinking을 끄고 한 번에 긴 글을 통째로 생성하면서 입력으로 분량을 채웠다. 그 결과 뒤로 갈수록 반복·일반론·부자연스러운 한국어가 늘었다. 생성을 **개요 → 섹션 → (검수)** 3단계로 끊고, 각 단계의 프롬프트를 입력·출력·제약까지 확정해 재현 가능하게 만든다.

## 2. 목표 (end goal)

- 동일 주제 입력 시 **구조가 일정하고**(H2 4~6개), 섹션 간 내용이 겹치지 않으며, 검수 게이트 통과율 80% 이상인 원고를 안정적으로 생성.
- 프롬프트만 보고 누구나 같은 품질을 재현. 모델 등급을 낮춰도 명세가 품질을 지탱.

## 3. 설계 / 결정

### 3.1 단계 분리 원칙

| 단계 | 모델 변수 | thinking | 출력 크기 | 입력 |
|---|---|---|---|---|
| 개요 | `MODEL_OUTLINE` | ON | 작게(≤600토큰) | 풍부하게(주제·타깃·톤·참고자료) |
| 섹션 | `MODEL_SECTION` | OFF | 섹션당 작게(≤700토큰) | 해당 섹션 정보만 |
| 검수 | `MODEL_REVIEW` | OFF, 저온 | 매우 작게(≤300토큰) | 제목+본문 |

근거: 추론이 필요한 건 "전체 구조 설계"(개요)뿐이다. 본문은 짧은 단위로 끊으면 추론 없이도 품질이 유지되고 비용·속도가 좋아진다.

### 3.2 핵심 제약(품질을 만드는 규칙)

- 섹션 본문은 **250~450자**, 같은 말 다른 표현 금지, **구체적 예시·수치 1개 이상** 포함.
- 소제목(H2) 줄은 본문에서 다시 쓰지 않는다(통합 시 코드가 붙임).
- 앞 섹션과 내용 중복 금지(섹션 프롬프트에 명시).
- 출력은 지정 형식만. 개요는 JSON, 본문은 마크다운 본문만.

## 4. 프롬프트 명세

### 4.1 개요 (OUTLINE)

입력 변수: `topic`, `audience`, `tone`, `references`

출력(JSON 고정):
```json
{"title": "...", "sections": [{"h2": "...", "point": "..."}]}
```

규칙:
- H2 4~6개, 각 H2에 핵심 메시지 1줄.
- 제목은 클릭 유도하되 과장/낚시 금지.
- 코드펜스·설명문 금지(파서가 첫 `{`~마지막 `}`만 추출하지만 깨끗하게).

검증: `json.loads` 성공 + `sections` 길이 4~6 + 각 항목에 `h2`,`point` 존재. 실패 시 1회 재생성, 재실패면 draft 유지(다음 주기).

### 4.2 섹션 (SECTION)

입력 변수: `title`, `h2`, `point`, `tone`

출력: 마크다운 본문만(H2 줄 제외).

규칙(위 3.2 제약). "앞 섹션과 겹치지 않게"를 매 호출에 명시. 필요 시 직전 섹션 요약을 입력에 추가하는 옵션 확장 가능(중복률이 높게 측정될 때).

### 4.3 검수 (REVIEW) — 상세는 02 문서

출력(JSON 고정):
```json
{"score": 0-100, "issues": ["..."], "verdict": "pass|fail"}
```
issues 키: `factual_doubt | generic | repetitive | off_topic | unnatural_ko`. score 80 미만이면 fail.

## 5. 골격 반영 사항

- `llm/prompts.py`: 위 제약을 시스템 프롬프트에 더 명시적으로 반영(예시 출력 포함, 중복 금지 강조).
- `pipeline/generate.py`: 개요 JSON 검증 로직 강화(섹션 수/필드 검증), 실패 시 재생성 1회.
- `config.py`: 단계별 토큰 상한을 설정값으로 노출(`MAX_TOKENS_OUTLINE/SECTION/REVIEW`).

(실제 수정은 CHANGELOG에 기록)
