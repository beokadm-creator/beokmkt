# [07] 검색 노출(SEO) 최적화 — 채널별 타깃 엔진

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `research/provider.py`, `research/collect.py`, `pipeline/seo.py`, `pipeline/generate.py`, `llm/prompts.py`, `config.py`, `db/schema.sql`
- 선행 문서: [05](./05-근거기반-콘텐츠엔진-재설계.md), [06](./06-유형별-템플릿-명세.md)

## 1. 배경 / 관건

"제대로 된 글"의 최종 목적은 **구글·네이버 검색에서 잘 걸리는 것**(노출)이다. 그런데 **구글 SEO와 네이버 SEO는 다른 게임**이다. 따라서 발행 채널별로 그 채널의 네이티브 검색엔진에 맞춰 최적화한다.

확정 방침: **채널별 타깃 엔진 분리.**

| 채널 | 타깃 엔진 | 최적화 기조 |
|---|---|---|
| 네이버 블로그 | 네이버 | C-Rank·D.I.A.+ : 주제 일관성·체류·사진/동영상·최신성·자연스러운 구성 |
| 티스토리 | 구글 | 표준 SEO·E-E-A-T·구조화 |
| 자체 블로그 | 구글 | 표준 SEO·E-E-A-T·구조화 |

## 2. 두 엔진의 차이 (왜 다르게 써야 하나)

### 구글
- 검색 의도 충족 + 신뢰(E-E-A-T) + 구조화가 핵심.
- 제목/메타/H2에 키워드, 명확한 정보 구조, FAQ/표 등 리치 요소, 출처 신뢰.
- 우리 근거기반 엔진(05)이 잘 들어맞음.

### 네이버
- 자체 알고리즘(C-Rank: 출처·전문성 누적 / D.I.A.+: 문서의 경험·구체성).
- **네이버 블로그 콘텐츠를 우대** → 네이버 노출엔 네이버 블로그 채널이 유리.
- 중시: 주제 일관성(한 블로그가 한 분야 집중), 사진/동영상, 체류시간, 최신성, 본인 경험·구체 디테일, 적절한 길이, 태그.
- 주의: 기계적·중복·과최적화(키워드 떡칠)는 오히려 누락(스팸 필터).
- 즉 **잘 쓴 글 ≠ 네이버 상위노출.** 경험·구체성·포맷(사진 위치)·일관성이 별도로 중요.

## 3. 엔진을 엔진 분석에 반영 (SERP 분석)

타깃 엔진의 실제 상위 결과를 분석해 의도·구성을 맞춘다.

- **네이버**: 네이버 검색 API(블로그/웹) → 상위 글 제목·요약 수집 → coverage_targets·키워드·구성 참고. (키 보유)
- **구글**: 구글계열 검색 공급자(Tavily 등) → 상위 결과 제목·요약.

분리 원칙(중요): **SERP 분석(노출용)과 사실 수집(근거용)은 분리한다.**
- SERP 분석 = 타깃 엔진(네이버/구글)에서 "무엇이 어떻게 노출되나".
- 사실 수집 = 본문이 풍부한 일반 웹(Tavily 등)에서 "검증 가능한 사실". 사실은 엔진과 무관하다.

## 4. 새 파이프라인 (05에 SEO 계층 추가)

```
주제(+유형+채널)
 → ① 의도/키워드
 → ②a SERP 분석   (타깃 엔진: 네이버 API / 구글)  → 경쟁 제목·구성·노출 키워드
 → ②b 사실 수집   (일반 웹: 근거용)
 → ③ 근거팩 (사실 + coverage_targets는 ②a 반영)
 → ④ 근거기반 개요(유형 템플릿 + 타깃 엔진 구조 힌트)
 → ⑤ 근거기반 섹션
 → ⑥ SEO 최적화   (엔진별: 제목/메타/태그/포맷)
 → ⑦ 사실검증
 → ⑧ 품질 검수
 → 발행
```

## 5. 엔진별 SEO 최적화 단계 (⑥)

LLM이 초안 + 주키워드 + 타깃 엔진을 받아 산출:

공통 산출물: `seo_title`, `meta_description`, `tags[]`, `primary_keyword 배치 점검`.

### 구글 최적화
- 제목 60자 내외, 주키워드 앞쪽. 메타 120~155자.
- H2에 키워드 변형 자연 배치, FAQ/요약/표 권장.
- 과최적화 금지(키워드 밀도 자연스럽게).

### 네이버 최적화
- 제목: 검색어 자연 포함 + 클릭 유도(낚시 금지).
- **본문에 사진/동영상 삽입 위치 마커**(`[이미지: ...]`) 권장 — 네이버는 미디어 포함 글 우대.
- 경험·구체 디테일 강조(D.I.A.+), 자연스러운 구어체 허용.
- 태그 5~10개(주제 일관). 키워드 떡칠·기계적 반복 금지(스팸 필터).
- 적정 길이(너무 짧지 않게), 최신성(발행 시 최신 정보 반영).

## 6. 골격 반영 사항

- `research/provider.py`: `NaverSearchProvider`(네이버 검색 API) 추가, `get_provider(engine)`로 엔진별 분기.
- `config.py`: `NAVER_CLIENT_ID/SECRET`, `CHANNEL_TARGET_ENGINE` 매핑, SERP/근거 공급자 분리 설정.
- `research/collect.py`: `analyze_serp(engine, keyword)` — 타깃 엔진 상위 제목/요약 수집.
- `research/evidence.py`: coverage 생성에 타깃 SERP 제목 사용.
- `pipeline/seo.py`: 엔진별 최적화(`optimize(engine, draft, ...)`).
- `pipeline/generate.py`: 채널→엔진 매핑, SERP 분석, SEO 단계 통합.
- `llm/prompts.py`: `SEO_GOOGLE`, `SEO_NAVER` 프롬프트.
- `db/schema.sql`: `target_engine`, `tags`, `seo_title` 저장.

## 7. 미해결 / 다음 단계

- 네이버 이미지 자동 생성/삽입(현재는 마커만) — 별도 기획.
- 네이버 블로그 지수(C-Rank)는 글 단위가 아니라 블로그 누적이라, **주제 일관 운영 정책**(한 블로그=한 분야) 필요 → 운영 문서로.
- 구글 구조화 데이터(JSON-LD, FAQ schema) 자동 삽입 — 자체 블로그부터.
- 키워드 난이도/검색량 데이터 소스 연동(상위노출 가능성 사전 판단).
