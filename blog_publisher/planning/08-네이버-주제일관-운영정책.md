# [08] 네이버 주제 일관 운영 정책 (C-Rank 대응)

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `config.py`, `db/schema.sql`, `db/db.py`, `run.py`
- 선행 문서: [07](./07-검색노출-SEO-최적화.md)

## 1. 배경

네이버 C-Rank는 **글 단위가 아니라 블로그(출처) 누적 지수**다. 한 블로그가 한 분야에 집중할수록 그 분야 검색에서 상위노출이 쉬워진다. 잡다하게 여러 분야를 섞으면 어느 분야에서도 지수가 안 쌓인다. 따라서 **블로그=분야 집중** 운영을 시스템이 강제해야 한다.

## 2. 목표

- 각 네이버 블로그가 1~2개 핵심 분야(주제 클러스터)에 집중한다.
- 시드(주제)가 분야에 맞는 블로그로 배정된다.
- 분야 이탈 글이 섞이지 않게 막는다.

## 3. 설계 / 결정

### 3.1 블로그 프로필

블로그(계정)마다 분야·카테고리·키워드 군을 정의한다.

```python
BLOG_PROFILES = {
  "naver_tech": {"channel": "naver", "blog_id": "...",
                 "category": "IT/가전", "keywords": ["이어폰","노트북","가전"]},
  "naver_money": {"channel": "naver", "blog_id": "...",
                  "category": "재테크", "keywords": ["신용점수","대출","청약"]},
}
```

### 3.2 시드 배정 규칙

- 시드 등록 시 `category`(또는 blog profile)를 지정/추론한다.
- 네이버 채널 시드는 반드시 한 프로필에 속한다. 프로필과 무관한 주제는 거부 또는 보류.
- posts에 `category`, `blog_profile`를 저장해 추적.

### 3.3 발행 일관성 운영

- 한 블로그의 발행은 그 분야 글로만 구성.
- 발행 빈도/최신성 유지(03 재고·스케줄과 연계): 분야별로 꾸준히.
- 분야 내에서 토픽 클러스터(허브-스포크) 구성 → 내부 링크로 연결(후속 SEO).

## 4. 골격 반영

- `config.py`: `BLOG_PROFILES` + `profile_for(category|topic)` 헬퍼.
- `db/schema.sql`: posts에 `category`, `blog_profile` 추가.
- `db/db.py`: insert_draft에 category/profile 인자.
- `run.py`: seed 명령에 category 인자, 네이버 시드의 프로필 일치 검증.

## 5. 미해결 / 다음 단계

- 주제→카테고리 자동 분류(LLM)로 배정 자동화.
- 허브-스포크 내부 링크 자동화(별도 SEO 문서).
- 블로그별 발행 쿼터/최신성 모니터링 지표(03 status_report 확장).
