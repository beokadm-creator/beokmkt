## 1. 배경 / 문제

"모델을 낮춰야 하나"를 감으로 정하던 것을 데이터로 바꾼다. 검수 게이트를 측정 도구로 삼아, 통과율을 기준으로 모델 등급을 조정한다. 동시에 발행 전 품질 하한선을 강제한다.

## 2. 목표 (end goal)

- 발행되는 모든 글이 정량 기준(길이/중복/구조)과 정성 기준(사실성/자연스러움)을 통과.
- 모델 등급 변경의 의사결정이 **통과율 수치**로 자동 정당화됨.
- 비용 대비 품질의 최적점을 단계별로 탐색 가능.

## 3. 설계 / 결정

### 3.1 2단 게이트

비용 순서가 핵심: 싼 규칙으로 먼저 거르고, 통과분만 비싼 LLM 평가.

```
규칙 게이트(무료)  ──fail──▶ draft 복귀(재생성)
       │pass
LLM 게이트(유료)  ──fail──▶ draft 복귀
       │pass
   reviewed
```

### 3.2 규칙 게이트 임계값 (config 노출)

| 항목 | 변수 | 기본값 | 의미 |
|---|---|---|---|
| 최소 본문 길이 | `MIN_BODY_LEN` | 800 | 가시 문자 수(마크다운 기호 제외) |
| 최대 중복률 | `MAX_DUP_RATIO` | 0.18 | 3-gram 중복 비율, 넘으면 늘려쓰기 |
| 최소 소제목 수 | `MIN_HEADINGS` | 3 | H2/H3 개수 |
| 금칙어 | `BANNED_WORDS` | (비움) | 채널 정책/브랜드 금지어 |

임계값은 실제 글로 1~2주 튜닝한다. 초기엔 느슨하게 시작해 거짓 탈락을 보며 조인다.

### 3.3 LLM 게이트 기준

- 출력: `{"score", "issues", "verdict"}` (01 §4.3).
- `MIN_REVIEW_SCORE`(기본 80) 미만 또는 verdict=fail → 탈락.
- 온도 0.0(재현성), 짧은 출력, thinking OFF.
- 검수기 자체 오류는 발행으로 새지 않게 **보류(draft 복귀)** 처리.

### 3.4 모델 등급 결정 절차 (감 → 데이터)

1. **기준선 측정**: 현재 모델 조합으로 N개(예 50개) 생성 → 게이트 통과율·평균 score 기록.
2. **단계적 하향**: 한 단계만 낮춘다(예: `MODEL_SECTION`만). 같은 N개 재생성 → 통과율 재측정.
3. **판정**: 통과율 하락 폭이 임계(예 5%p) 이내면 채택, 초과면 원복.
4. **단계 독립**: 개요·섹션·검수를 따로 실험(개요는 품질 민감 → 마지막에 건드림).
5. **기록**: 각 실험을 별도 기획 문서(`NN-모델실험-YYYYMMDD.md`)로 남기고 결론을 CHANGELOG에 반영.

판단 지표: `통과율`, `평균 score`, `건당 비용`, `건당 소요시간`.

## 4. 프롬프트 명세 (해당 시)

검수 프롬프트는 01 §4.3에 확정. 본 문서는 임계값/운영 절차를 관장.

## 5. 골격 반영 사항

- `config.py`: 임계값들은 이미 노출됨. 모델 실험용 메모 주석 추가.
- `pipeline/review.py`: 통과/탈락 카운트를 로그로 남겨 통과율 집계 가능하게(후속).
- 측정 스크립트 `tools/measure_passrate.py`(후속): N개 생성→게이트→통과율/비용 리포트.
