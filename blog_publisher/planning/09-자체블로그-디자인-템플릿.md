# [09] 자체 블로그 디자인 / 템플릿 개선

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `render/template.html`, `render/style.css`, `render/renderer.py`, `publishers/selfhosted.py`
- 선행 문서: [07](./07-검색노출-SEO-최적화.md)

## 1. 배경

자체 발행 블로그가 **사람이 읽기 좋고(가독성)** + **검색이 잘되는 구조(SEO)**로 나와야 한다. 지금은 본문만 API로 넣을 뿐 구조/디자인이 없다. 발행 시 일관된 HTML 템플릿으로 렌더링한다.

## 2. 목표

- 읽기 좋은 타이포그래피·여백·반응형(모바일 우선).
- 검색 친화 구조: 시맨틱 HTML5, 메타/OG/트위터 카드, schema.org Article JSON-LD, 목차(TOC), 빠른 로딩(인라인 CSS, 외부 의존 최소).

## 3. 설계 / 결정

### 3.1 가독성

- 본문 폭 제한(680~720px), 큰 본문 폰트(17~19px), 줄간격 1.7.
- 명확한 제목 위계(h1=제목, h2/h3 본문), 충분한 섹션 여백.
- 시스템 폰트 스택(웹폰트 로딩 비용 회피), 다크모드 대응(prefers-color-scheme).

### 3.2 SEO 구조

- `<article>`, `<header>`, `<nav>`(TOC), `<time>`, `<footer>` 시맨틱 태그.
- `<head>`: title, meta description, canonical, OG(title/description/image/type=article), Twitter card.
- **JSON-LD**: Article(headline, datePublished, author, mainEntityOfPage, keywords).
- 본문 H2에서 자동 목차 생성(앵커 링크) → 체류·구조 신호.
- 이미지 `alt`, lazy loading. 출처 링크는 `rel="nofollow"` 옵션.

### 3.3 렌더러

- 입력: post(title, body_markdown, meta_desc, tags, published_at, canonical_url, author).
- 마크다운 → HTML 변환 + H2 앵커/TOC 삽입 + 템플릿 치환.
- 출력: 완성 HTML 문자열. selfhosted 발행 시 이 HTML을 사용.

## 4. 골격 반영

- `render/template.html`: 시맨틱+SEO 헤드+TOC 슬롯이 있는 단일 템플릿.
- `render/style.css`: 가독성·반응형·다크모드 CSS(인라인 삽입용).
- `render/renderer.py`: 마크다운 변환, TOC 생성, JSON-LD 구성, 템플릿 렌더.
- `publishers/selfhosted.py`: 본문 대신 렌더된 HTML 전송(옵션).

## 5. 미해결 / 다음 단계

- 마크다운 변환기 선택(markdown 패키지) vs 자체 경량 변환. 초기엔 경량.
- 대표 이미지(og:image) 생성/지정 정책(기획 07 이미지와 연계).
- 카테고리/태그 아카이브 페이지, 사이트맵·RSS(블로그 단위 SEO).
- AMP/코어웹바이탈 최적화.
