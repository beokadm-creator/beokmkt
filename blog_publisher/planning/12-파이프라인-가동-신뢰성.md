# [12] 파이프라인 가동 신뢰성 (최우선)

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `llm/parse.py`, `llm/prompts.py`, 전 `pipeline/*`·`research/*`, `tools/selftest.py`
- 선행 문서: [00 GOAL](./00-GOAL-방향과-개발내용.md), 01·05

## 1. 배경 / 최우선 목표

기능 추가보다 **파이프라인이 끝까지 안 끊기고 도는 것**이 먼저다. 자동 파이프라인이 깨지는 가장 흔한 지점은 **LLM의 구조화 출력(JSON) 파싱**이다(코드펜스, 군더더기 텍스트, 트레일링 콤마, 스마트따옴표 등). 여기를 견고하게 만들고 프롬프트를 그 기준에 맞춘다.

## 2. 목표 (측정 가능)

- 모든 JSON 단계가 흔한 오염(코드펜스/설명문/트레일링콤마/스마트따옴표)에도 파싱된다.
- 1회 자동 복구 재시도로 일시적 형식 오류를 흡수한다.
- 한 글의 한 단계 실패가 전체 배치를 죽이지 않는다(이미 워커별 try/격리, 유지).
- **키 없이도** 전 파이프라인을 mock으로 끝까지 돌려 검증할 수 있다(`selftest`).

## 3. 설계 / 결정

### 3.1 견고한 JSON 파싱 계층 (`llm/parse.py`)

- `extract_json(raw)`:
  1) 코드펜스(```json ... ```) 제거,
  2) 첫 `{`/`[`부터 **괄호 균형 스캔**으로 정확한 JSON 조각 추출(문자열 내 괄호 무시),
  3) 실패 시 트레일링 콤마 제거·스마트따옴표 정규화 후 재시도,
  4) 그래도 실패면 명확한 오류(원문 일부 포함).
- `chat_json(llm, system, user, **kw)`: LLM 호출 + 파싱. 실패 시 "유효 JSON만 출력" 지시를 덧붙여 **1회 재시도**.

모든 모듈의 임시 `_json`/`_extract_json`을 이 계층으로 통일한다.

### 3.2 프롬프트 정렬 (JSON-only 표준)

- 구조화 출력 프롬프트는 공통 말미 규칙을 둔다: **"유효한 JSON 하나만 출력. 코드펜스/주석/설명/앞뒤 텍스트 금지."**
- 스키마를 프롬프트에 명시(이미 됨). thinking은 구조화 출력 단계에서 OFF(혼합 출력 방지), 의도/개요만 ON.

### 3.3 오프라인 셀프테스트 (`tools/selftest.py`, `run.py selftest`)

- mock LLM(단계별 결정적 JSON/텍스트) + mock 검색 + mock 발행으로
  `generate → factcheck → review → schedule → publish`를 실제 코드로 관통.
- 한 글이 `published`에 도달하면 PASS. 회귀 가드 + 키 없이 가동 점검.

## 4. 골격 반영

- `llm/parse.py` 신규(extract_json/chat_json).
- `research/evidence.py`, `pipeline/{generate,factcheck,review,seo}.py`: parse 계층 사용.
- `llm/prompts.py`: JSON-only 말미 규칙 정렬.
- `tools/selftest.py` + `run.py selftest`.
