# planning — 기획 문서 인덱스

이 폴더는 블로그 발행 시스템의 **모든 기획 산출물**을 보관한다.
여기 쌓인 문서들이 최종 목표(end-goal) 프롬프트 개발의 근거가 된다.

## 작업 규칙 (고정)

1. **기획은 전부 문서로.** 새로 기획하는 내용은 예외 없이 이 폴더에 번호가 붙은 기획 문서로 저장한다. 채팅으로만 끝내지 않는다.
2. **번호 규칙.** `NN-제목.md` 형식. 번호는 순차 증가(`01-`, `02-`…). 한 번 부여한 번호/파일명은 바꾸지 않는다(참조 안정성).
3. **골격 수정은 기록.** 코드·설정 등 골격을 직접 수정하면 [`CHANGELOG.md`](./CHANGELOG.md)에 "날짜 / 대상 파일 / 무엇을 / 왜"를 남긴다.
4. **템플릿 사용.** 새 기획 문서는 [`_TEMPLATE.md`](./_TEMPLATE.md)에서 복사해 시작한다.
5. **상태 표기.** 각 문서 상단에 `상태: 초안 | 확정 | 보류 | 폐기`를 둔다. 확정된 문서만 프롬프트 개발에 반영한다.

## end-goal 프롬프트와의 연결

확정된 기획 문서 → 해당 단계(생성/검수/발행)의 프롬프트 명세로 변환 →
`llm/prompts.py` 등 실제 골격에 반영(반영 시 CHANGELOG 기록).
프롬프트 자체의 버전 기록도 별도 기획 문서로 관리한다.
