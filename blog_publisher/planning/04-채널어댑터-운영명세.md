# [04] 채널 어댑터 운영 명세

- 상태: 확정
- 작성일: 2026-06-14
- 관련 골격: `publishers/base.py`, `publishers/selfhosted.py`, `publishers/naver.py`, `publishers/tistory.py`, `config.py`
- 선행 문서: [DESIGN.md](../DESIGN.md) §4, [03](./03-발행스케줄링-안정화-정책.md)

## 1. 배경 / 문제

채널마다 안정성이 다르다. 자체 블로그는 API로 안정적이고, 네이버·티스토리는 공식 글쓰기 API가 없어 Playwright로 에디터를 조작해야 하며 본질적으로 깨지기 쉽다. 발행 워커가 채널 내부를 모르도록 인터페이스를 고정하고, 불안정 채널의 깨짐을 줄이는 운영 규칙을 확정한다.

## 2. 목표 (end goal)

- 새 채널 추가가 `Publisher` 구현 + 레지스트리 등록으로 끝난다.
- 네이버/티스토리 깨짐을 줄이고, 깨져도 흐름을 멈추지 않는다(사람 폴백).
- 모든 실패가 `RetryableError`/`FatalError`로 분류되어 03 정책에 따라 처리된다.

## 3. 설계 / 결정

### 3.1 공통 인터페이스

```python
class Publisher(Protocol):
    name: str
    def publish(self, post) -> str: ...   # 성공: URL / 실패: Retryable|Fatal
```

발행 워커는 `PUBLISHERS[channel].publish(post)`만 호출. 채널 분기 없음.

### 3.2 자체 블로그 (신뢰 채널)

- REST API로 직접 POST. 계약:
  - 요청: `POST {SELFHOST_API_URL}/posts`, `Authorization: Bearer ...`, body `{title, content, status}`.
  - 응답: 200/201 + `{url}`. 401/403 → Fatal, 5xx → Retryable, 그 외 4xx → Fatal.
- 가장 먼저 안정화. 신규 주제는 자체 블로그로 먼저 발행해 파이프라인 검증.

### 3.3 네이버 / 티스토리 (불안정 채널)

깨짐 최소화 규칙(확정):

1. **세션 재사용**: 최초 1회 수동 로그인 → `storage_state`(`naver_state.json`/`tistory_state.json`) 저장 → 매 발행은 이 세션 로드. 매번 로그인 금지(차단 위험).
2. **사람형 동작**: 타이핑 딜레이 20~80ms, 발행 시각 지터(03), headless 기본 off 권장.
3. **셀렉터 단일 출처**: 각 어댑터 상단 `SELECTORS` 딕셔너리에 집중. DOM 변경 시 여기만 수정.
4. **단계 검증**: 에디터 진입·발행 버튼 활성 등 각 단계 확인 실패 시 `RetryableError`.
5. **세션 만료 처리**: 로그인 화면으로 튕기면 `FatalError`로 던져 세션 갱신 알림(자동 재시도로 풀리지 않음).
6. **폴백**: 재시도 소진 시 `needs_human` → 사람이 수동 발행.

### 3.4 셀렉터 갱신 운영

- `SELECTORS`는 예시값. 실제 발행 전 1회 점검 필수.
- 갱신 시 CHANGELOG에 "어느 채널 / 어느 셀렉터 / 왜(=DOM 변경)" 기록.
- 점검 도구 `tools/inspect_editor.py`(후속): 브라우저 열어 현재 셀렉터 유효성 확인.

### 3.5 세션 저장 도구 (필수 선행)

`tools/login.py`(후속): Playwright로 브라우저를 띄워 사용자가 직접 로그인 → `context.storage_state(path=...)` 저장. 네이버/티스토리 각각.

## 4. 골격 반영 사항

- `publishers/naver.py`, `tistory.py`: 세션 만료 감지 시 `FatalError` 분기 보강(후속).
- `tools/login.py`, `tools/inspect_editor.py` 신규(후속).
- `config.py`: 채널별 `*_HEADLESS`, `*_USER_AGENT`, `*_BLOG_ID` 이미 노출됨.
